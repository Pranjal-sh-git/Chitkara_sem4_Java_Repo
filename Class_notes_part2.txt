Array in Java:
==============

--An array is used to store multiple values of the same data type in a single variable.

Example:


int[] arr = new int[5]; //5 int variable is created with thier default value as an array obj.



--The above created array is capable of storing 5 integers.


--In Java, arrays are treated as objects.

--Array object is created in heap memory

--Array variable is a reference variable

--Array class is automatically created by JVM (It is a special type of class)

--Inside an object there is a non-static variable is availabe called "length" which represents the size of that array.

	public final int length


Example:


int[] arr = new int[5];

System.out.println(arr); // [I@...

System.out.println(arr[0]); // 0
System.out.println(arr[3]); // 0
System.out.println(arr.length); // 5
System.out.println(arr[6]); // ArrayIndexOutOfBoundsException

void calculateStatistics(Student[] students) {

    if (students == null || students.length == 0) {
        System.out.println("No student data available.");
        return;
    }

    double total = 0;
    Student topper = students[0];

    for (Student s : students) {
        total += s.getMarks();

        if (s.getMarks() > topper.getMarks()) {
            topper = s;
        }
    }

    double average = total / students.length;

    System.out.println("Average Marks: " + average);

    System.out.println("Highest Marks Student:");
    System.out.println("Roll: " + topper.getRoll());
    System.out.println("Name: " + topper.getName());
    System.out.println("Marks: " + topper.getMarks());
}

void searchStudent(Student[] students, int roll) {

    boolean found = false;

    for (Student s : students) {
        if (s.getRoll() == roll) {
            System.out.println("Student Found:");
            System.out.println("Name: " + s.getName());
            System.out.println("Marks: " + s.getMarks());
            found = true;
            break;
        }
    }

    if (!found) {
        System.out.println("Student not found.");
    }
}

Jagged Array:
============


A Jagged Array is a multi-dimensional array where each row can have different number of columns.

In simple terms:
Each row can store a different number of elements.


Example:

Classroom Seating arrangement:

Row1: 3 students
Row2: 5 students
Row3: 2 students






Why Jagged Array?
-----------------


Used when data size is not uniform, for example:

--Students having different numbers of subjects

--Employees having different projects

--Monthly sales with varying days

--Different course modules


Using a normal 2D array wastes space whereas Jagged arrays fit perfectly.


Important Concept
------------------


--In Java, a 2D array is actually an array of arrays.

--So each row can be created with different sizes.

Syntax:

int[][] arr = new int[3][];

Here:

Total rows = 3

Column sizes not defined yet


Assign Different Column Sizes


int[][] arr = new int[3][];

arr[0] = new int[2];
arr[1] = new int[4];
arr[2] = new int[3];



Example Program
----------------

public class Demo {
    public static void main(String[] args) {

        int[][] arr = new int[3][];
		
		
		arr[0] = new int[] {10,20};

		}

}

--It uses Dual-Pivot QuickSort internally
 
--Modifies original array


Sorting in decending order:


 	 int[] arr = {5, 1, 9, 3, 7};

        // Step 1: Sort in ascending order
        Arrays.sort(arr);

        // Step 2: Reverse array
        int start = 0;
        int end = arr.length - 1;

        while (start < end) {
            int temp = arr[start];
            arr[start] = arr[end];
            arr[end] = temp;

            start++;
            end--;
        }

        // Step 3: Print result
        System.out.println(Arrays.toString(arr));

Inheritance in Java:
=====================


--Inheritance is a mechanism in Java where one class acquires the properties and behaviors of another class.

In simple words:

--Getting properties and methods of one class into another class.

Example

--If Dog is an Animal, then Dog inherits properties of Animal.--with the help of inheritance we achieve IS-A realtionship



Advantages of Inheritance:
--------------------------

1. Code reusability

2. Runtime polymorphism

3. Better code organization

4. Easier maintenance



Relationships in Java:
----------------------

--In Java, there are 2 types of relationships exists:

1. Has-A (Assosication)
	--Composistion (Strong Has-A)
	--Aggregation (Weak Has-A)




Has-A relationship  example:

class A {

B b1 = new B();

}


Here A class contains the object of another class B


2. Is-A relationship: (Inheritance)


--lets assume we need to develop a forest project, in which we need to create some class like:

Dog	Cat 	Tiger	Lion ,20 


and all classes should have following methods:


sleep() //1000
walk() //5000
eat() //6000
makeNoise()



--In this situation, we need to repeat the above methods inside the multiple classes, to avoid this situations we can make use of Inheritance concept in Java.

--in which we can create a Parent class called Animal and keep all the common functionalities inside this class:

--And then all the above classes can extends the Animal class, and become the child class of this Animal class, later on any of the above classes can override the makeNoise method, and define its own functionlities also.


--we can put one more category inside this Inherince hirarchey, like one as WildAnimal and another as DomesticAnimal also.



Type of inheritance:
=================

Note: At class level java supports only multilevel inheritance(single, hirarical).where as with the help of interface we can achieve multiple inheritance.

-----------------------

Access modifiers:
==============

--it specifies the accessebility of a class or its members outside that class or package.

we have 4 types of access modifiers:

1.public  : accessebiltiy is global (other package or other classes also)

2.default: it is similar to public as long as inside the same package, but outside the package it works as private

3.protected: it is simillar to default, but outside the package we can access it by using inheritance.

4.private : the accessebility is restricted to that class only.


Note: A normal class/outer-class can only be either default or public but that class members (variables, methods, constructors, inner classes) can be private ,public, default, protected.

Note: we can not apply access modifiers to the local variables.



--default constructor given by java compiler is always public.

--if we make constructor of a class a default then we can not create its object or extends this class outside that package.

--if we make constructor of a class a private then we can not create its object or extends this class outide that class even in the same package also.

--"protected" gives extra access to subclasses even in different packages.

--Rectangle

abstract class Shape {
    protected String shapeName;
    protected String color;

    Shape(String shapeName, String color) {
        this.shapeName = shapeName;
        this.color = color;
    }

    final void showDetails() {
        System.out.println("Shape: " + shapeName + ", Color: " + color);
    }

    abstract void calculateArea();
}

class Circle extends Shape {
    double r;

    Circle(String color, double r) {
        super("Circle", color);
        this.r = r;
    }

    void calculateArea() {
        System.out.println("Area: " + (3.14 * r * r));
    }
}

class Rectangle extends Shape {
    int l, b;

    Rectangle(String color, int l, int b) {
        super("Rectangle", color);
        this.l = l;
        this.b = b;
    }

    void calculateArea() {
        System.out.println("Area: " + (l * b));
    }
}

public class Demo1 {
    public static void main(String[] args) {

        Shape s1 = new Circle("Red", 5);
        Shape s2 = new Rectangle("Blue", 4, 6);

        s1.showDetails();
        s1.calculateArea();

        System.out.println();

        s2.showDetails();
        s2.calculateArea();
    }
}

Interface in Java:
==================



In Java there are 3 types of valid structures are available:

1. Full implemented structure: Normal or concrete class

2. Partially implemented structure: Abstract class

3. Full unimplemented structure: interface // till Java 7




from Java 8 onwards we can place method with body also inside an interface (default method or static method).
where from Java 9 onwards we can place a private method also (with body) inside an interface.



***Note: With the help of an interface we achive loose coupling in Java.

--Class is a blueprint for an object, where as Interface is also like blueprint of the class.

 
--Inside an interface if we place any method wituout body, that method will be public and abstract by default wheter we mention it or not.

public class AnagramExample {

    public static void main(String[] args) {

        String s1 = "listen";
        String s2 = "silent";

        if(isAnagram(s1, s2))
            System.out.println("Strings are Anagram");
        else
            System.out.println("Strings are NOT Anagram");
    }

    static boolean isAnagram(String s1, String s2){

        if(s1.length() != s2.length())
            return false;

        int[] count = new int[256];

        for(int i=0;i<s1.length();i++){
            count[s1.charAt(i)]++;
            count[s2.charAt(i)]--;
        }

        for(int x : count)
            if(x != 0)
                return false;

        return true;
    }
}





Length check	O(1)
Loop over string	O(n)
Second loop check	O(256)

public String reverseEachWord(String s1) {

        StringBuilder word = new StringBuilder();
        StringBuilder result = new StringBuilder();

        for(int i = 0; i < s1.length(); i++)
        {
            char ch = s1.charAt(i);

            if(ch != ' ')
            {
                word.append(ch);
            }
            else
            {
                result.append(word.reverse()).append(" ");

        }

Exception Handling in Java:
===========================

What is an Exception?

--An Exception is an object created by the JVM that represents a runtime error (logical error) occurring during program execution.

Definition

--An exception is an unwanted or unexpected event that disrupts the normal flow of a program.



there r two types of error in java

1.syntax error:-improper environment and syntactical error.it results in compilation error.


2.logical error (Abnormal condition):-sometime jvm would be unable to execute statements in the bytecode for some reasons this results in logical error.


---whenever a logical error occurs program is terminated and control comes out of the program unconditionally without mentioning from which part of the program it has come out.

ex:-
class A{
public static void main(String[] args) {
	int i=100;
	int j=0;
	int x=i/j;
	System.out.println(x);
	System.out.println("End of the main");
}
}

---the above program will be perfectly compliled,but while executing it jvm terminates the program abnormally.because....

---there r some predefined classes in java whose objects r created by jvm whenever it encounters a logical errors.Names of such classes r found under the Exception class in API.

--for each and every logical error(abnormal situation) there is a corresponding exception class in the java.lang package.

---whenever jvm encounters a logical error,it creats an objects of the coresponding exception class and since this this object is not explicitly handled this object reaches back to the jvm ,jvm accept this object and terminets the program .

--now if we r able to handle those exception class object,we can prevent the program from getting termineted abnormally.

---thus concept of handling the exception class object and avoiding them from reaching back to the jvm is known as Exception Handling.



--we can handle the exception class object by assigning them to the corresponding class refference..

----we have two keyword try and catch to implement exception handling.

--the try keyword recognizes the exception class objects created by jvm and it get holds those objects and transfer them to the catch block.
--in catch block we should be able to assign those objects to the appropriate reff so that they will not reach to the jvm and therefore program would not be terminated.

3.we r not allowing the jvm to terminate the entire program. only the try block in which the error has occured would be terminated.

finally Keyword in Java
======================


The finally block is used with a try block to ensure that certain statements are always executed, whether an exception occurs or not.


--The finally block contains statements that are guaranteed to execute after the execution of the try and catch blocks.




Syntax:
--------

try {
    // risky code
}
catch(Exception e) {
    // exception handling
}
finally {
    // cleanup code (always executes)
}


Need of the finally Block:
-------------------------

Sometimes some statements inside a try block must be executed compulsorily, regardless of whether an exception occurs or not.

These statements are usually related to resource cleanup, such as:

--Closing files

--Closing database connections

--Closing network connections

--Releasing system resources


Example Problem Without finally:
-------------------------------

try {
    // get connection to printer
    connectPrinter();

    printDocument();

    closeConnection();
}
catch(Exception e) {
    System.out.println(e);
}

Problem:

--Suppose an exception occurs during: printDocument();

--Then the program immediately jumps to the catch block.

Therefore:

closeConnection(); will not execute.

--The printer resource remains allocated but not released.

--This leads to ineffective resource management.

Possible Solution (Not Recommended):
------------------------------------

We may try writing the cleanup code outside the try-catch.

try {
    connectPrinter();
    printDocument();
}
catch(Exception e) {
    System.out.println(e);
}

closeConnection();

Although this works, it is not recommended.


Why This Approach is Not Recommended

It breaks good coding principles.

1. Integrity

--All statements related to a specific task should remain within the same logical block.

--Moving statements outside breaks the logical grouping of code.

2. Modularity

--The code becomes mixed with unrelated statements, making it difficult to understand and maintain.

--There is no proper separation of logic.


Correct Solution Using finally
---------------------------

try {
    connectPrinter();
    printDocument();
}
catch(Exception e) {
    System.out.println(e);
}
finally {
    closeConnection();
}

--Now the closeConnection() statement will always execute, whether an exception occurs or not.

Important Notes:

1. A try block can have: 

--multiple catch blocks
--only one finally block

Example:

try {

}
catch(IOException e) {

}
catch(ArithmeticException e) {

}
finally {

}


2. finally is optional

--But if used, only one finally block is allowed.


3. A try block can exist without a catch block, if it has a finally block.

Example:

try {
    System.out.println("Hello");
}
finally {
    System.out.println("Finally block");
}

Output

Hello
Finally block

--However, this approach is generally not recommended, because exceptions remain unhandled.



Real-Life Analogy: 
----------------

--Think of finally block as cleaning the room after work.

Even if:

--work finishes successfully

--work fails due to some problem

--You still clean the room before leaving.

Common Uses of finally
----------------------

finally is commonly used for:

--Closing files

--Closing database connections

--Releasing network sockets

--Releasing locks

Example:


FileReader fr = null;

try {
    fr = new FileReader("data.txt");
}
catch(Exception e) {
    System.out.println(e);
}
finally {
    if(fr != null)
        fr.close();
}



throw keyword in Java:
======================

--The throw keyword in Java is used to manually create and throw an exception from our program.

--Normally, exceptions are created by JVM automatically, but sometimes we want to create our own exception based on some condition, in that case we use throw.

Definition:

--throw is used to explicitly throw an exception from a method or block of code.

Syntax:
-------

	throw new ExceptionClass("Error message");

Example:

	throw new ArithmeticException("Number cannot be divided by zero");


Example:



class Test
{

	static void withdraw(int balance, int amount){
    		
		if(amount > balance){
        		
			throw new RuntimeException("Insufficient balance");
    		
		}

    	System.out.println("Withdraw successful");

}


    public static void main(String args[])
    {
  
	withdraw(10000,12000);
  
    }
}

Here,

--JVM did not create exception. We manually created exception and propagating to the user using throw keyword.

--Since we are throwing an unchecked exception, we need not intimate the caller using function signature, but if we throw a checked exception in that case we need to mention the caller also by using throws keyword with the method signature. and in order to call that method caller must either need to wrap that call inside the try and catch block or the caller can also delegate the exception handling duty to its caller using throws keyword.

example:

 


class Test
{

	static void withdraw(int balance, int amount) throws Exception{
    		
		if(amount > balance){
        		
			throw new Exception("Insufficient balance");
    		
		}

    	System.out.println("Withdraw successful");

}


    public static void main(String args[])
    {
  	try{
	
		withdraw(10000,12000);

  	}catch(Exception e){

		e.printStackTrace();
	}

    }
}

User-Defined exception:
======================

Introduction

So far, we have used predefined exception classes such as:

ArithmeticException

NullPointerException

ArrayIndexOutOfBoundsException

IOException

--The objects of these exceptions are created automatically by the JVM when a predefined logical error occurs.

Example:

int x = 10 / 0;

JVM automatically creates:

--ArithmeticException


What is a User-Defined Exception?
------------------------------

--Sometimes the programmer wants to define their own logical rules.

--If those rules are violated, we may want to generate our own exception.

--These exceptions are called User-Defined Exceptions (Custom Exceptions).

Definition

--A user-defined exception is a custom exception class created by the programmer to represent application-specific errors.

Example situations:

Invalid age

Insufficient balance

Invalid password

Invalid marks


Important Point:
---------------

--A user-defined logical error is not considered an error by the JVM.

--Therefore the programmer must explicitly create and throw the exception.


Steps to Create a User-Defined Exception
-------------------------------------

Step 1

--Create a class that extends Exception

Step 2


--Define a parameterrized constructor inside that class by taking String as an argument, and pass that String to its parent class constructor.

//or

--Override the toString() or getMessage() method



Step 3

--Create an object of that exception

Step 4

--Throw it using the throw keyword


Example: Invalid Age Exception

Step 1: Create Custom Exception Class

class InvalidAgeException extends Exception {

    InvalidAgeException(String msg) {
        super(msg);
    }

}


Step 2: Create a Class That Uses This Exception


class Emp {

    public int getPension(int age, int sal) throws InvalidAgeException {

        int pen = 0;

        if(age > 45 && age < 100) {
            pen = (age * sal) / 100;
        }
        else {
            InvalidAgeException ae = new InvalidAgeException();
            throw ae;
        }

        return pen;
    }

}


Step 3: Use the Method in main()

class Test {

    public static void main(String[] args) {

        Emp e = new Emp();

        try {
            int p = e.getPension(30, 50000);
            System.out.println("Pension = " + p);
        }
        catch(InvalidAgeException ex) {
            System.out.println(ex);
        }

    }
}




Password checking example:
-------------------------



InvalidPasswordException.java:
------------------------------

class InvalidPasswordException extends Exception {
    InvalidPasswordException(String message) {
        super(message);
    }
}



Demo.java:
----------


import java.util.Scanner;

class Demo {

    static void checkPassword(String password) throws InvalidPasswordException {
        
        if(password.length() < 8) {
            throw new InvalidPasswordException("Password must contain at least 8 characters");
        }

        boolean hasDigit = false;

        for(int i = 0; i < password.length(); i++) {
            if(Character.isDigit(password.charAt(i))) {
                hasDigit = true;
                break;
            }
        }

        if(!hasDigit) {
            throw new InvalidPasswordException("Password must contain at least one digit");
        }

        System.out.println("Password is valid");
    }

    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        System.out.print("Enter password: ");
        String pwd = sc.nextLine();

        try {
            checkPassword(pwd);
        } 
        catch(InvalidPasswordException e) {
            System.out.println("Exception: " + e.getMessage());
        }
    }
}




Constructor Throwing Checked Exception:
------------------------------------


Rule

--If a constructor of a class declares a checked exception, then creating an object of that class must either:

--handle the exception using try-catch, or

--declare the exception using throws.

--Because of this, we cannot directly create the object as an instance member of another class without handling the exception.

Example

class A {

    A() throws Exception {
        System.out.println("Constructor of A");
    }

}

Now consider another class:

class B {

    A a1 = new A();   // Compile Time Error

}


Why Error?

A() throws a checked exception, but instance variable initialization cannot handle the exception directly.

Compiler error:

Unhandled exception type Exception


Correct Approach
----------------

Create the object inside the constructor of the class.

class B {

    A a1;

    B() throws Exception {
        a1 = new A();
    }

}

Or handle it using try-catch.


Java 7 Features:
---------------

1. Single Catch Block for Multiple Exceptions (Java 7 Feature)

It allows one catch block to handle multiple exceptions.

Syntax

Exceptions are separated using the pipe symbol |.

catch(ClassNotFoundException | IOException e) {
    e.printStackTrace();
}


Example

import java.io.*;

class Demo {

    public static void main(String[] args) {

        try {

            int x = 10 / 0;

            Class.forName("Test");

        }
        catch(ArithmeticException | ClassNotFoundException e) {

            System.out.println(e);

        }

    }

}


Important Point

--In a multi-catch block, the exception variable becomes implicitly final.

--So we cannot reassign it.

Example (Not Allowed):

catch(IOException | ClassNotFoundException e) {

    e = new IOException();   // Compile-time error

}


Another Rule

--We cannot catch exceptions from the same hierarchy together.(no parent-child relationships)

Example (Invalid):

catch(Exception | IOException e) {

}


IOException -> child of Exception

So IOException is already covered by Exception, making the second one redundant.

This produces a compile-time error.



but this is possible:

try {
    // risky code
}
catch(SQLException | IOException e) {
    System.out.println("Common Error handling for IOE and SQLE");
}
catch(Exception e) {
    System.out.println("Other Error handling..");
}



2. Try-With-Resources


--try-with-resources is a special try statement used to automatically close resources after execution.

--A resource is an object that must be closed after use.

Examples:

File streams

Database connections

Network connections

Scanner objects


Syntax


try(Resource resource = new Resource()) {

    // code using resource

}
catch(Exception e) {

}

Java automatically calls: resource.close(); when execution finishes.


Example:

import java.io.*;

class Demo {

    public static void main(String[] args) {

        try(FileInputStream fis = new FileInputStream("data.txt")) {

            System.out.println("File opened");

        }
        catch(Exception e) {
            System.out.println(e);
        }

    }

}


Requirement for Try-With-Resources:


The object must implement: java.lang.AutoCloseable Interface.

AutoCloseable Interface
-----------------------

It contains only one method:

public void close() throws Exception;

Any class implementing this interface can be used in try-with-resources.

Example:


FileInputStream

BufferedReader

Scanner

Connection
